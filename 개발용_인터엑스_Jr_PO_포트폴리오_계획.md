# 인터엑스 Jr. PO 포트폴리오 계획

목표: 웹 포트폴리오 한 페이지와 5페이지 PDF가 같은 메시지를 말하게 만든다.

핵심 메시지:

> AI를 말로만 쓰는 기획자가 아니라, AI 도구로 직접 제품을 만들고 배포하며 복잡한 업무와 데이터를 사용자가 판단 가능한 제품 구조로 바꿔본 Jr. PO입니다.

## 1. 웹 포트폴리오 구조

형태:

- 한 페이지 스크롤
- 중앙 본문: 포트폴리오 케이스 스터디
- 우측 고정 패널: OpenAI API 기반 대화형 챗봇
- 디자인 톤: OpenAI 스타일 레퍼런스 기반, 흰 캔버스/검정 텍스트/회색 보조 텍스트/최소 색상

첫 화면:

- 제목: 복잡한 업무와 데이터를 사용자가 판단할 수 있는 제품 구조로 바꿉니다.
- 서브카피: 데블록에서는 이레링크와 시픈의 기획/PM을 맡았고, 퇴사 후에는 Dimetric, 스톡스토커, 사주중독 등을 단독으로 만들었습니다.
- CTA: Resume PDF, StockStalker, Dimetric GitHub, GitHub
- 채팅 입력 예시:
  - 인터엑스 Jr. PO와 가장 잘 맞는 경험은?
  - 이레링크에서 정확히 뭘 했어?
  - 스톡스토커가 AI Native 증거인 이유는?

## 2. 웹 섹션 순서

### 1. INTERX Fit

4개 키워드만 보여준다.

- AI Native: ChatGPT/Claude/Codex 기반 구현, 오류 분석, 문서화, AI 결과 검증 흐름 설계
- B2B/Admin: 예약, 매칭, 정산, 매출분석, 상품/리뷰 관리
- Data/Product Logic: 수집, 정규화, 캐싱, 스코어링, Graph Patch 검증, 비교 UX
- Execution: PRD, IA, 화면설계, 일부 디자인, 개발 매니징, QA

### 2. StockStalker

역할: 초기 사용 지표가 있는 데이터 제품 사례

넣을 증거:

- 2026.06 - 2026.06
- 100% 단독 기획/개발/배포
- 배포 후 1주 동안 약 1,000명 방문, 12,000건 종목 조회
- SEC/DART 공시 데이터 실시간 제공, 룰베이스 요약 업데이트
- KIS/yfinance, Supabase snapshot cache, 서버 메모리 캐시
- 링크: https://stock-khaki.vercel.app
- GitHub: https://github.com/LAF-labs/stock

말해야 할 핵심:

- 주식 앱을 만든 것이 아니라, 불완전하고 서로 다른 데이터 소스를 사용자가 판단 가능한 UX로 정리한 경험이다.

### 3. Dimetric

역할: 대표 AI/CAD 제품 구조 사례

넣을 증거:

- 2026.04 - 2026.06
- 단독 기획/개발
- GitHub: https://github.com/LAF-labs/dimetric
- 스크린샷:
  - `assets/dimetric-editor-gallery.png`: 채팅 패널, 3D 캔버스, 어셈블리/파트 패치 패널이 함께 보이는 화면
  - `assets/dimetric-editor-canvas.png`: 채팅 패널과 3D 캔버스 중심 화면
- 문제 발견: 대학/지자체 3D 프린터 운용률이 5% 미만이고, 니즈는 있으나 도안 제작(CAD) 허들이 높아 사용하지 못하는 상황
- 로그인 후 프로젝트 보관함에서 프로젝트 생성, 열기, 삭제
- 프로젝트 카드: 미리보기, 이름, 수정일, 스냅샷 개수
- 편집 화면: 왼쪽 채팅 패널, 중앙 Three.js 3D 캔버스, 오른쪽 수치/기능 패널
- AI 권한 모드: AI가 변경 가능한 파라미터 범위 제어
- AI 결과를 바로 적용하지 않고 구조화된 Graph Patch로 검증
- CAD 컴파일 검증 통과 후 모델 갱신 및 스냅샷 생성
- Python, build123d 등을 CAD engine으로 활용
- CAD 워커 기반 STL, 3MF, STEP, GLB 내보내기
- 출력성 검사: 최소 두께, 베드 크기, 오버행 등 3D 프린팅 위험 요소를 한국어 경고로 안내

말해야 할 핵심:

- AI가 만든 결과를 무조건 반영하는 도구가 아니라, 실패한 AI 응답과 CAD 변경이 원본 프로젝트를 망가뜨리지 않도록 검증/스냅샷/내보내기 흐름을 설계한 경험이다.

### 4. IreLink

역할: 실무 기획 메인 사례

넣을 증거:

- 2025.05 - 2026.04
- 외국인 의료관광 B2B 운영 시스템
- 공개 가능 화면: 어드민 PC 2장, 모바일 앱 1장
- PRD, IA, 화면설계서, Figma 일부 디자인, QA 시트
- 레퍼런스 수집, 기획, 개발 매니징, 일부 디자인, QA
- 엑셀/칠판 기반 업무의 디지털 전환
- 클라이언트 업무 시간 단축 피드백
- 매출 데이터 오류 발견

말해야 할 핵심:

- 러프한 요구사항을 제품 요구사항, 화면, Admin, 앱 구조로 바꾼 경험이다.

### 5. Shipn / Tastit

역할: 아이디어가 회사 내부 프로젝트로 이어진 사례

넣을 증거:

- 시픈: 2026.01 - 2026.04, 내부 프로젝트
- 담당: 아이디어 제안, 기획, 일부 디자인, 개발 매니징, GA4 작업

말해야 할 핵심:

- 대학 창업에서 검증한 문제의식이 입사 후 내부 프로젝트 제안으로 이어졌다.

## 3. 챗봇 설계

목표:

- 채용담당자가 페이지를 읽다가 바로 질문할 수 있게 한다.
- 답변은 김기범의 이력/프로젝트 근거 안에서만 하게 한다.
- 모르는 내용은 추측하지 않고 확인 필요라고 답하게 한다.

추천 질문:

- 김기범의 핵심 강점을 3줄로 요약해줘.
- 이레링크에서 어떤 역할을 했어?
- 인터엑스 Jr. PO에 왜 맞아?
- 제조 경험이 부족한 점은 어떻게 보완돼?
- Dimetric에서 AI 결과를 어떻게 검증했어?
- 스톡스토커는 토이 프로젝트가 아니라는 증거가 뭐야?

API 원칙:

- 브라우저에 OpenAI API 키를 넣지 않는다.
- 서버의 `/api/chat`에서 `OPENAI_API_KEY`를 읽는다.
- API 키가 없을 때는 로컬 fallback 답변을 보여준다.

## 4. 5페이지 PDF 포트폴리오

웹 내용을 압축해 PDF로도 제출한다.

### Page 1. Candidate Snapshot

- 김기범 | AI Native Builder형 Jr. Product Owner
- 핵심 문장 1개
- 증거 4개: AI Native, B2B/Admin, Data/Product Logic, Startup Execution
- 링크: 웹 포트폴리오, StockStalker, Dimetric GitHub, GitHub

### Page 2. Dimetric

- 문제: AI가 생성한 CAD 변경을 검증 없이 적용하면 원본 프로젝트를 망가뜨릴 수 있다.
- 역할: 단독 기획/개발
- 구조: Chat -> Graph Patch 검증 -> CAD 컴파일 검증 -> 모델 갱신 -> 스냅샷
- 화면: 프로젝트 보관함, 채팅 패널, Three.js 3D 캔버스, 수치/기능 패널
- 출력: STL, 3MF, STEP, GLB 내보내기와 출력성 경고

### Page 3. IreLink

- 문제: 엑셀/칠판 기반 의료관광 운영
- 역할: PRD, IA, 화면설계, 일부 디자인, QA, 개발 매니징
- 산출물: 어드민 PC 2장, 모바일 앱 1장
- 결과: 업무 시간 단축 피드백, 매출 데이터 오류 발견

### Page 4. StockStalker

- 문제: 데이터가 흩어져 있고 판단 흐름이 끊긴다.
- 역할: 단독 기획/개발/배포
- 구조: API -> cache -> score -> compare UX
- 결과: 배포 후 1주 1,000명 방문, 12,000건 종목 조회, SEC/DART 업데이트
- 하단 보조: 사주중독, 스드맵, 시픈/테이스팃 한 줄 요약

### Page 5. INTERX Fit

표로 정리한다.

| 인터엑스 요구 | 증거 |
|---|---|
| AI Native | Dimetric의 AI 변경 검증 흐름, 스톡스토커 단독 제작, Codex/Claude/ChatGPT 활용 |
| A-to-Z | 기획, 개발 협업, QA, 배포 |
| B2B/Admin | 이레링크 Admin, 사주중독/스드맵 Admin |
| 데이터/제품 구조화 | Graph Patch/CAD 검증, 주식/의료관광/리뷰/취향 데이터 |
| 제조 도메인 보완 | 낯선 도메인의 업무 흐름을 빠르게 구조화한 경험 |
