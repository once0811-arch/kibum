# 인터엑스 회사/제품 리서치

작성일: 2026-06-17  
목적: 인터엑스 제조 AI Solution PO 사전과제용 웹 포트폴리오의 메시지, 섹션, 시각화 기준을 잡기 위한 리서치.

## 1. 회사 한 줄 정의

인터엑스는 산업 AX & 자율제조 전문 파트너다. 공식 홈페이지는 인터엑스를 `Enterprise AX & Autonomous Manufacturing Partner`, `AX 자율제조 솔루션 전문기업`으로 소개한다.

핵심 문장:

- 제조 데이터 표준화와 SDF 기반 자율제조 시스템을 바탕으로 산업 디지털 전환 및 AI 자율제조 토탈 솔루션을 제공한다.
- AI와 빅데이터 분석을 기반으로 제조공정 최적화 기술과 자율제조 운영 서비스를 제공한다.
- 단순 데이터 나열/시각화를 넘어 생산 의사결정, 빠른 실행, 문제 개선, 이슈 사전 대응을 가능하게 하는 방향을 지향한다.

포트폴리오 해석:

- 인터엑스가 원하는 PO는 단순 화면 기획자가 아니라, 데이터/운영/AI를 의사결정 가능한 시스템으로 구조화하는 사람이다.
- 따라서 김기범 포트폴리오의 중심 언어는 `예쁜 UI`, `기획 참여`가 아니라 `복잡한 데이터와 운영을 사용 가능한 제품 구조로 전환`이어야 한다.

## 2. 제품/서비스 체계

인터엑스 공식 제품군은 크게 다음 축으로 정리된다.

| 제품군 | 공식 핵심 | 포트폴리오 연결점 |
|---|---|---|
| AI & DT 솔루션 | 생산조건 최적화, 품질 예측, 품질 검사, 산업 안전, 제조 디지털 트윈 | Dimetric의 AI/CAD 검증 흐름, StockStalker의 데이터 분석/스코어링, ireh link의 운영 대시보드 |
| AI Kit 솔루션 | 제조 현장에서 실시간 데이터 수집, 분석, AI 의사결정을 실행 | 현장 적용성과 운영자 UX를 이해하는 역량 강조 |
| AX 인프라 솔루션 | 제조 프로세스의 디지털화, 데이터 표준화, AI 모델 개발 서버 | ireh link의 엑셀/수기 업무 표준화, 데이터 마이그레이션 오류 발견 경험 |
| 생성형 AI 솔루션 | 제조 전문 지식 기반 Gen.AI, 자연어 AI 소통, 업무 자동화 | AI Native 역량, Dimetric의 자연어 기반 CAD 생성/수정, 사주중독/StockStalker 개발 방식 |
| 지능형 공급망 솔루션 | 데이터 수집/저장/관리/분석, 산업 시스템 간 실시간 데이터 교환 | StockStalker의 수집/정규화/캐싱/비교 UX, 데이터 상태를 사용자가 이해하게 하는 설계 |

## 3. 세부 제품에서 뽑아야 할 키워드

### Recipe.AI - 생산조건 최적화 AI

- 복잡한 제조 공정을 수집 데이터 기반 AI 분석으로 최적 생산 조건을 제시한다.
- 문제 키워드: 품질 일관성 부족, 데이터 단절, 의사결정 비효율, 숙련 인력 부족, 디지털 전환 미흡.

포트폴리오 반영:

- `복잡한 조건을 추천/검증 가능한 구조로 바꾸는 역량`을 Dimetric의 Graph Patch/CAD Compile 흐름으로 보여준다.

### Quality.AI - 품질 예측 최적화 AI

- 제조 공정 데이터를 분석해 품질 저하 요인을 사전 감지하고 최적 조건을 제안한다.
- 문제 키워드: 사전 예측 어려움, 실시간 모니터링 부재, 휴먼 에러, 다운타임, 표준화 부족.
- Demo 설명에는 실시간 공정 데이터 수집/분석, 품질 예측/분류, 품질 기준 최적화, PQCD 향상이 언급된다.

포트폴리오 반영:

- StockStalker의 점수화/비교/상태 표시를 `데이터 기반 판단 UX`로 해석한다.
- ireh link의 매출/예약/정산 Dashboard는 `운영 상태를 한눈에 파악하게 만든 경험`으로 배치한다.

### Inspection.AI - 품질 검사 AI

- AI 이미지 분석으로 제품 표면 결함을 신속·정확하게 검사한다.
- 검사자 숙련도/컨디션에 좌우되던 품질을 개선하는 방향.

포트폴리오 반영:

- 사전과제에서 직접 이미지 검사 경험을 과장하지 않는다.
- 대신 `AI 결과를 사람이 신뢰할 수 있게 검증/표시하는 구조`를 Dimetric으로 연결한다.

### Safety.AI - 산업 안전/환경 AI

- 객체 감지 AI와 위험 구역 모니터링, 작업자 경고 알림으로 실시간 사고 예방을 제공한다.

포트폴리오 반영:

- 실시간 경고/상태 표시 경험은 StockStalker의 API 상태, 데이터 확인, refresh cooldown 구조로 연결 가능하다.

### Manufacturing.DT - 제조 디지털 트윈

- AI 솔루션 연계를 통한 자율제조 통합 운영 서비스.
- AI 기반 가상 모델로 제조 공정을 실시간 모니터링하고 최적화한다.

포트폴리오 반영:

- Dimetric의 Three.js 기반 3D 캔버스와 CAD 워커, 스냅샷 흐름은 `디지털 모델을 보고 조작하는 경험`으로 가장 강하게 연결된다.

### Gen.AI - 제조 특화 생성형 AI

- 제조 전문 지식 기반 솔루션으로, 사용자가 프로그래밍 언어 지식 없이 자연어로 AI와 소통할 수 있다.
- 문제 키워드: 지식 격차, 수작업 중심 업무, 데이터 검색 어려움, 인터페이스 접근성, 데이터 일관성 부족.
- 태그: Chatbot Assistant, AI Agent, 구매/발주 자동화, RAG, 보고서 자동 작성, 내부 데이터 연계.

포트폴리오 반영:

- Dimetric을 첫 번째 주요 사례로 둔다. 자연어 → 구조화된 변경안 → 검증 → 반영 구조가 Gen.AI와 가장 직접적으로 맞다.
- 채팅 모달은 단순 부가기능이 아니라 `AI Native 포트폴리오`의 증거로 둔다.

### Document.AI

- 문서 구조와 데이터 유형을 고려해 LLM/RAG에 최적화된 형태로 파싱하고 JSON으로 병합한다.

포트폴리오 반영:

- 김기범의 문서화 역량(PRD, IA, 화면설계서, QA 시트)을 `문서 산출물`로만 두지 말고, `개발자가 실행 가능한 구조화`로 해석해야 한다.

### AI Kit

- 제조 현장에서 실시간 데이터 수집, 분석, AI 의사결정을 실행하도록 설계된 솔루션.
- 기존 PLC/SCADA 등 IT 시스템과 쉽게 연동, 엣지/클라우드 확장, 센서·카메라 데이터 실시간 분석, 사전 학습 모델, 직관적 설정/운영 지원.

포트폴리오 반영:

- `현장 적용성`, `운영자 UX`, `직관적 설정` 키워드를 ireh link의 Admin 설계와 연결한다.

### AX 인프라

- 데이터 수집 및 서비스 개발을 위한 플랫폼, 제조 데이터 표준화/관리, AI 모델 개발을 위한 서버를 제공한다.

포트폴리오 반영:

- ireh link의 수기/엑셀 업무 표준화와 데이터 마이그레이션 오류 발견은 `데이터 인프라 감각`으로 보여줄 수 있다.

### 지능형 공급망 솔루션

- 다양한 데이터를 수집, 저장, 관리, 분석하고 가치를 창출하는 환경을 제공한다.
- 제조 데이터 표준 기반으로 산업 시스템 간 실시간 데이터 교환을 지원한다.

포트폴리오 반영:

- StockStalker의 데이터 수집/정규화/캐싱/비교 UX를 `분산 데이터의 신뢰 가능한 읽기 경험`으로 시각화한다.

## 4. 채용 공고와 연결되는 핵심 요구

Wanted 공고상 Jr. PO 충원 포지션은 다음을 명시한다.

- 제조업을 몰라도 좋지만, 기획에 대한 집요함과 PO 실행력을 요구한다.
- 경력보다 문제 해결 집요함, 추상적 아이디어를 구체화하는 논리적 설계력, AI 툴을 활용한 차세대 PO 역량을 중요하게 본다.
- 필수/핵심 역량: 서비스 기획 또는 PO/PM 1년 이상, A-to-Z Experience, AI Native, Communication.
- UX/UI 담당 미션은 `복잡한 제조 데이터를 가장 직관적으로`다.

포트폴리오 결론:

- 첫 화면은 “제조 경험은 없지만 배우겠습니다”가 아니라, `복잡한 데이터/AI/운영을 이미 제품으로 바꿨다`가 되어야 한다.
- 제조 도메인 부족은 결핍으로 숨기지 말고, `도메인 지식보다 문제 구조화와 AI Native 실행력을 보는 Jr. 포지션`이라는 공고 문장에 맞춘다.
- Dimetric, ireh link, StockStalker 순서가 가장 설득력 있다.
