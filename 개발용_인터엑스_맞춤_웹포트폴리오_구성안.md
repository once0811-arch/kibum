# 인터엑스 맞춤 웹 포트폴리오 구성안

작성일: 2026-06-17  
목표: 인터엑스 제조 AI Solution PO 사전과제에서 김기범의 역량과 업무 경험을 `웹 경험`으로 설득한다.

## 1. 한 줄 전략

제조 경험이 아니라, `복잡한 데이터·운영·AI 응답을 사용자가 판단 가능한 제품 구조로 바꾸는 사람`임을 증명한다.

## 2. IntegratedBio 벤치마킹 방식

IntegratedBio는 다음 방식을 사용한다.

- 첫 화면을 긴 sticky hero로 잡고, 스크롤 진행에 따라 배경/문장/진행 번호가 바뀐다.
- 큰 타이포가 메시지 자체를 그래픽처럼 사용한다.
- 01/02/03 구조로 플랫폼의 핵심 구성요소를 증명한다.
- 카드와 이미지가 clip-path로 드러나며, 장표가 아니라 하나의 긴 내러티브처럼 이어진다.
- Newsroom/Publication 영역처럼 실제 증거를 편집해 신뢰를 만든다.

포트폴리오 적용:

- 01: AI Native - Dimetric
- 02: Admin/Dashboard - ireh link
- 03: Data UX - StockStalker
- 하단 Evidence: sippn, 자투리, 테이스팃, SajuHook 수치

## 3. 최종 사이트 흐름

### Section 0. Sticky Hero

목적: 10초 안에 핵심 포지셔닝 전달.

카피:

- H1: `복잡한 데이터와 AI를 제품으로 출고하는 Jr. PO`
- 보조: `인터엑스가 찾는 AI Native, A-to-Z, Admin/Dashboard, Communication 역량을 실제 제품으로 증명합니다.`

스크롤 단계:

| Step | 배경 | 메시지 |
|---|---|---|
| 01 | Dimetric 3D 캔버스 | 자연어 요청을 검증 가능한 CAD 변경으로 바꿨습니다 |
| 02 | ireh link Admin | 러프한 요구사항을 PRD/IA/SB/QA로 바꿨습니다 |
| 03 | StockStalker 차트 | 분산된 데이터를 판단 가능한 비교 UX로 바꿨습니다 |

### Section 1. INTERX Fit Map

목적: 인터엑스 제품군과 후보 경험을 즉시 연결.

구성:

| INTERX 키워드 | 김기범 증거 | 시각화 |
|---|---|---|
| Gen.AI / 자연어 AI | Dimetric 자연어→Graph Patch→CAD Compile | pipeline chart |
| Manufacturing.DT / Physical.AI | Three.js 3D 모델, CAD export | 3D viewport mock |
| Admin/Dashboard UX | ireh link 예약/매출/정산 Admin | operation map |
| Data Space / 데이터 표준화 | StockStalker API/cache/scoring/compare | data flow diagram |
| A-to-Z Experience | 기획, 디자인 일부, 개발 매니징, QA, 배포 | lifecycle ring |

### Section 2. Proof Numbers

목적: 사전과제 요구사항인 수치/도표 중심 구성.

반드시 차트화할 수치:

- StockStalker: 1주 1,000명 방문, 12,000건 조회
- SajuHook: 초기 매출 20만 원
- Jaturi: 81개 업체 방문, 80여 개 PoC, 4분기 매출 1,057만 원, 거래건수 +30.4%p, 배송 이용률 +11.3%p, 정부지원 4,628만 원 + 6,000만 원
- Tastit: 700명 설문, 600명 MVP 테스트, 구매 의향 82%, 제휴 검토 7곳, 확정 3곳
- Dimetric: 3D 프린터 운용률 5% 미만 문제에서 출발, STL/3MF/STEP/GLB export
- Deblock: ireh link 2025.05-2026.04, sippn 2026.01-2026.04

시각화:

- bubble field: 숫자 크기로 임팩트 비교
- horizontal bar: 퍼센트/건수
- timeline: 경력 흐름
- lifecycle ring: A-to-Z 범위

### Section 3. Case 01 - Dimetric

목적: AI Native와 개발 베이스 우대사항을 가장 강하게 증명.

구성:

- Problem: 3D 프린터 운용률 5% 미만, CAD 허들
- System: Chat → Graph Patch → CAD Compile → Snapshot → Export
- UX: 왼쪽 채팅, 중앙 3D Canvas, 오른쪽 수치/기능 패널
- Risk Control: 실패한 AI/CAD 변경이 원본을 망가뜨리지 않도록 검증
- Evidence: GitHub 링크, 스크린샷

### Section 4. Case 02 - ireh link

목적: B2B/Admin/Dashboard와 Communication 증명.

구성:

- Problem: 클라이언트 요구사항이 러프하고 레퍼런스가 없음. 기존 업무는 엑셀/칠판/수기.
- Role: 레퍼런스 수집, PRD, IA, 화면설계서, Figma 일부 디자인, QA 시트, 개발 매니징.
- System: 예약/매칭, 매출분석, 차트관리, 상품판매, 인센티브 정산, 데이터 대시보드, 웹빌더.
- Evidence: Admin 예약 화면, 매출 화면, 모바일 앱 화면.

### Section 5. Case 03 - StockStalker

목적: 데이터 직접 추출/분석 가능성을 증명.

구성:

- Problem: 종목 데이터가 여러 API/시장/공시로 분산되어 있음.
- System: KIS/yfinance, Supabase snapshot cache, memory cache, partial response, refresh cooldown.
- UX: 상세 분석, 비교, 가격 흐름, 핵심 숫자, 공시/뉴스.
- Result: 1주 1,000명 방문, 12,000건 조회, SEC/DART 공시와 룰베이스 요약 추가.

### Section 6. Operator Evidence

목적: 우대사항인 초기 스타트업 경험 증명.

구성:

- Jaturi: 방문 설문, PoC, 매출, 정부지원, 프로세스 개선 수치.
- Tastit/sippn: 창업 아이디어 검증에서 내부 프로젝트 제안까지 연결.
- SajuHook: 혼자 서비스 출시 후 초기 매출.

### Section 7. Closing

목적: 인터엑스 입사 후 포부를 과장 없이 정리.

카피 방향:

- 제조 지식은 빠르게 흡수한다.
- 대신 이미 갖춘 것은 문제 구조화, AI Native, Admin/Dashboard 설계, 데이터 UX, 출시 실행력이다.
- 인터엑스의 제조 데이터와 AI 모델을 현장 사용자가 이해하고 판단하는 화면으로 바꾸겠다.

## 4. 구현 체크리스트

- sticky hero는 유지하되 H1과 내러티브를 더 인터엑스 직접형으로 바꾼다.
- `INTERX Fit Map`을 새로 추가한다.
- 수치 전용 섹션을 추가하고 모든 수치를 차트화한다.
- 각 주요 case는 `Problem / System / Evidence / Result` 구조로 통일한다.
- 현재의 proof cards는 더 강한 데이터/제품 매칭 섹션으로 흡수하거나 축소한다.
- 채팅 모달은 유지하되 시스템 컨텍스트도 새 구조에 맞춘다.
- 모바일보다 데스크톱 몰입감을 우선한다.

## 5. 금지할 것

- “이력서는 프로젝트 단위로 읽혀야 합니다” 같은 내부 분석 문장.
- 역량 키워드 표만 나열하는 화면.
- PDF 장표처럼 섹션을 끊어 붙이는 구성.
- 이력서에 없는 수치 창작.
- 제조 경험 부족을 지나치게 방어적으로 표현하는 문장.
